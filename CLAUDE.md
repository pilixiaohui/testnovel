## 提示词与文档修改规范

修改 `orchestrator/memory/prompts/*.md` 或其他面向代理的提示词/文档时，必须遵守以下规范：

### 禁止补丁式说明

代理运行时不了解代码修改历史，补丁式说明会造成困惑。

**禁止使用**：
- `【新增】`、`【修改】`、`【删除】` 等变更标记
- `NOTE: xxx 已移除`、`原 xxx 代码已删除` 等历史说明
- `2.1.`、`5.1.` 等插入式编号（应重新整理为连续编号）
- `向后兼容`、`旧版本` 等暗示历史变更的措辞

**正确做法**：
- 直接写最终状态，不解释"之前是什么"
- 使用连续编号（1, 2, 3...）
- 内容应完整自洽，无需了解修改历史即可理解

### 保持结构完整

- 每个章节应有清晰的标题层级
- 规则列表应有明确的优先级顺序
- 避免在文档中间插入孤立的说明段落

### 示例

**错误**：
```markdown
## 决策规则
1. 规则 A
2. 规则 B
2.1. 【新增】规则 B 的补充  ← 补丁式编号和标记
3. 规则 C
```

**正确**：
```markdown
## 决策规则
1. 规则 A
2. 规则 B
   - 规则 B 的补充内容
3. 规则 C
```
