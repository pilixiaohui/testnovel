## 任务目标
彻底修复 SimulationView 的快速失败问题

## 问题分析
REVIEW 报告显示 SimulationView 中 startSimulation 函数仍存在 catch 块：
```ts
.catch(() => {
  errorMessage.value = 'Failed to start simulation.'
})
```
这违反了「快速失败/禁止兜底」约束。catch 块吞掉了异常，导致错误不会向上传播。

## 修复要求
文件: `project/frontend/src/views/SimulationView.vue`

修复方案（二选一）：
1. **完全移除 catch 块**：让 Promise rejection 自然传播，由上层错误边界处理
2. **改为 rethrow**：在 catch 中设置 errorMessage 后重新抛出错误

推荐方案 1（完全移除 catch），因为更符合快速失败原则。

## 验收标准
必须依次执行以下命令并在报告中汇报每项结果：
1. `grep -c '\.catch' project/frontend/src/views/SimulationView.vue` → 应为 0（无 catch 块）
2. `grep -c 'errorMessage' project/frontend/src/views/SimulationView.vue` → 应为 0 或仅用于声明（不在 catch 中使用）
3. `cd project/frontend && npm run build` → 构建成功
