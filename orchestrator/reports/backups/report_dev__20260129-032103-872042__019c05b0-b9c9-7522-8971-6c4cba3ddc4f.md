## 任务理解

工单要求实现的功能：  
1. 补齐 editor 组件目录与 6 个编辑器相关组件。  
2. 组件满足基础渲染与交互（编辑/预览/对比/日志/上下文）。  
3. 通过验收命令与构建。  

## 实现方案

### 设计思路
采用最小可用组件实现，避免过度设计（KISS/YAGNI），复用已有类型与组件（DRY），保证每个组件职责单一（SOLID-S），不添加兜底逻辑（快速失败）。

### 改动说明

| 文件 | 修改内容 | 实现逻辑 |
|-----|---------|---------|
| `project/frontend/src/components/editor/SceneEditor.vue:1` | 新增场景编辑器 | 使用 textarea 编辑内容，保存/取消通过 emit 通知外部 |
| `project/frontend/src/components/editor/ContentRenderer.vue:1` | 新增内容渲染器 | 支持 text/markdown 两种渲染，markdown 采用轻量转换 |
| `project/frontend/src/components/editor/VersionDiff.vue:1` | 新增版本对比 | 行级对比旧/新内容，差异行高亮显示 |
| `project/frontend/src/components/editor/MarkdownPreview.vue:1` | 新增 Markdown 预览 | 复用 ContentRenderer 渲染 markdown |
| `project/frontend/src/components/editor/SimulationLogs.vue:1` | 新增推演日志 | 展示回合列表与行动/裁决摘要 |
| `project/frontend/src/components/editor/SceneContextPanel.vue:1` | 新增场景上下文 | 展示摘要、期望结果、语义状态、实体/关系、前后场景 |

### 关键实现逻辑
- 数据流向：父组件通过 props 传入内容/上下文，子组件仅负责渲染与 emit 事件。  
- 关键处理：  
  - `VersionDiff` 以行级对比计算差异并标记 `changed`。  
  - `ContentRenderer` 在 markdown 模式下将基础语法转换为 HTML 后渲染。  
  - `SimulationLogs` 对回合行动与裁决结果做摘要展示。  
- 异常处理策略：不加兜底，直接按输入渲染。  
- 与其他模块交互：严格依赖 `SimulationRoundResult`、`SceneContextView` 类型。
