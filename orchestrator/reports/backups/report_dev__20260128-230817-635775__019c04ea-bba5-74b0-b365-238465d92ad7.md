## 实现方案

### 设计思路
提供最小化的 `gqlalchemy` 本地桩模块，覆盖 schema 单测所需的 `Node` 与 `Relationship` 基类，同时对 `Memgraph`/`Connection` 进行快速失败的占位实现，保证缺失依赖时立即报错而非隐式兜底。

### 改动说明

| 文件 | 修改内容 | 实现逻辑 |

| `project/backend/gqlalchemy/__init__.py:1` | 新增 Node/Relationship/ Memgraph 桩实现 | 满足 schema 导入与继承检查，运行时使用即快速失败 |
| `project/backend/gqlalchemy/connection.py:1` | 新增 Connection 桩实现 | 兼容 `from gqlalchemy.connection import Connection` 导入 |

### 关键实现逻辑
- 数据流向：schema 导入 `gqlalchemy` → 使用本地桩类完成继承/构造检查。
- 关键步骤：实现 `Node/Relationship` 接收 `**kwargs` 并保存 `_properties`；`Memgraph/Connection` 在被实际调用时抛出错误。
- 异常处理策略：不做兜底，缺失真实依赖时使用即报错（快速失败）。
- 与其他模块的交互方式：仅影响 `gqlalchemy` 导入路径，不改动业务逻辑或外部依赖。
