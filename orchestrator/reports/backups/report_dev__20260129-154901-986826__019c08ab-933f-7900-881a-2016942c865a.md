## 任务理解

工单要求实现的功能：
1. 修复 `HomeView.vue`，用 Element Plus 展示项目概览与快速入口，替换骨架文本
2. 校验路由与 `RouterView` 渲染，并在布局中集成 `AppHeader` 与 `AppSidebar`
3. 保证首页访问 `http://localhost:5173/` 时显示完整界面

## 实现方案

### 设计思路
以最小改动建立全局布局（Header + Sidebar + Main），HomeView 作为内容区展示项目概览与入口卡片；抽取导航数据避免重复，保持结构清晰并确保响应式。

### 改动说明

| 文件 | 修改内容 | 实现逻辑 |
|-----|---------|---------|
| `project/frontend/src/App.vue:1` | 增加应用壳布局，集成 AppHeader/AppSidebar/RouterView | 保证所有路由页统一布局与响应式结构 |
| `project/frontend/src/views/HomeView.vue:1` | 用 Element Plus 重写首页结构 | 展示项目概览、空状态与快速入口卡片 |
| `project/frontend/src/components/common/AppSidebar.vue:1` | 实现侧边导航菜单 | 复用导航数据与路由高亮 |
| `project/frontend/src/components/common/AppHeader.vue:1` | 抽离导航数据与高亮逻辑 | Header 与 Sidebar 共享导航配置，避免重复 |
| `project/frontend/src/utils/navigation.ts:1` | 新增导航常量与 active 解析 | DRY 抽取导航数据与活跃路由规则 |
| `project/frontend/src/main.ts:1` | 引入 Element Plus 与全局样式 | 统一 UI 风格并移除默认 Vite 样式干扰 |

### 关键实现逻辑
- 数据流向：HomeView 通过 `useProjectStore` 读取项目状态，决定展示项目概览或空状态；点击快速入口统一走 `router.push`。
- 关键处理步骤：抽取 `navigationItems` 与 `resolveActivePath`，Header/Sidebar 共享导航数据与激活逻辑。
- 异常处理策略：不做兜底与容错逻辑，依赖现有路由与组件渲染（符合快速失败原则）。
- 与其他模块交互：App.vue 负责布局包裹 RouterView，HomeView 作为内容区渲染，Header/Sidebar 作为导航入口。

## 总结、反思与展望
- 已完成的核心任务：首页重构、全局布局集成、导航结构统一与样式对齐，满足首页完整界面展示要求。
- KISS：布局层级保持简单（Header + Sidebar + Main）；HomeView 使用标准 Element Plus 组件组合。
- YAGNI：未引入额外状态与 API，避免超范围功能扩展。
- DRY：抽取 `navigationItems` 与 `resolveActivePath` 复用导航数据。
- SOLID：App.vue 只负责布局，HomeView 专注内容，导航常量独立管理。
- 遇到的挑战：默认 Vite 样式导致布局居中与颜色冲突，已通过切换全局样式统一解决。
- 下一步计划与建议：接入真实项目列表与最近项目数据（基于 store/api），再细化 Sidebar 分区与项目切换。
